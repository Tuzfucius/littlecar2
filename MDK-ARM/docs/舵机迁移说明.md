# 总线舵机迁移说明

## 1. 参考来源
当前总线舵机设备层参考旧项目：

- 仓库路径：`E:\littleCar\25省赛\10.9`
- 核心参考文件：`Core/Src/ft_servo.c`
- 真实底层发送函数：`Core/Src/main.c` 中的 `Servo_SendPositionAccelSpeedCommand()`

## 2. 已废弃的参考线
旧项目中的 `fashion_star_uart_servo` 相关内容已经废弃，本工程不再以它作为舵机实现基线。

当前工程的舵机模块只参考 `ft_servo.c` 实际使用的协议帧格式，不再迁移 `fashion_star_uart_servo.h/.c` 的收发结构。

## 3. 旧项目中可迁移的内容
从旧项目中保留并迁入当前工程的只有“设备层协议发送”这一部分：

- 帧头：`0xFF 0xFF`
- 指令：`0x03`
- 写入起始地址：`0x29`
- 数据区：
  - `acceleration`：1 字节
  - `position`：2 字节
  - `time`：2 字节，旧实现固定为 `0`
  - `speed`：2 字节
- 校验方式：
  - 从 `ID` 开始到最后一个数据字节逐字节求和
  - 最终对和按位取反

当前工程将这一逻辑封装为：

- `Core/Inc/bus_servo.h`
- `Core/Src/bus_servo.c`

## 4. 旧项目中不迁移的内容
以下内容属于流程层或机构位姿层，不进入当前设备层：

- `servo_blue`
- `servo_red`
- `servo_green`
- `servo_open`
- `servo_close`
- `servo_main_front`
- `servo_move_back`
- `near_catch`
- `far_catch`

这些函数本质上依赖具体机械结构标定值、动作时序以及其他执行机构联动，不应放进总线舵机底层模块。

## 5. 当前工程接入方式
当前工程固定采用：

- 串口：`UART4`
- 引脚：`PA0 / PA1`
- 设备层接口：
  - `BusServo_Init()`
  - `BusServo_StartReceive()`
  - `BusServo_SetPosition()`
  - `BusServo_SetPositionEx()`
  - `BusServo_SendCommand()`
  - `BusServo_SendGroup()`

其中：

- `BusServo_SetPositionEx()` 对齐旧项目的 `Servo_SendPositionAccelSpeedCommand()`
- `BusServo_SetPosition()` 是默认参数封装
- `BusServo_SendGroup()` 当前仅按顺序逐条发送，不定义协议级同步写

## 6. 当前设计边界
当前版本是“纯发送设备层”：

- 已实现：发包、参数检查、UART4 接入、单字节中断重启
- 未实现：回读、状态缓存、零位查询、闭环监控、协议级同步回包解析

如果后续确认当前这套舵机协议还需要读寄存器或回包解析，再在 `bus_servo.c` 上扩展接收状态机，不从已废弃的 `fashion_star` 方案回退。
