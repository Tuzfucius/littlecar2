# 一、整体任务架构

建议采用三层结构：

```text
硬件驱动层 BSP
    ↓
设备协议层 Device Driver
    ↓
应用任务层 App
```

对应关系：

```text
CubeMX生成代码
    ↓

    ↓
Driver：张大头电机、总线舵机、WIT陀螺仪、OPS、Jetson协议
    ↓
App：底盘控制、任务调度、安全保护、状态上报
```

# 三、各代码文件职责

## 1. CubeMX 自动生成文件

这些文件主要不要乱改，除非在 `USER CODE` 区域中写代码。

| 文件                    | 作用                            |
| --------------------- | ----------------------------- |
| `main.c`              | 系统初始化、主循环入口                   |
| `usart.c / usart.h`   | USART1/2/3、UART4/5、USART6 初始化 |
| `dma.c / dma.h`       | DMA 初始化                       |
| `gpio.c / gpio.h`     | GPIO 初始化                      |
| `tim.c / tim.h`       | TIM6/TIM7 等定时器初始化             |
| `stm32f4xx_it.c`      | 中断服务函数入口                      |
| `stm32f4xx_hal_msp.c` | 底层 MSP 初始化                    |

你自己写的业务逻辑不要堆在这些文件里。

---

## 2. `bsp_uart.c/h`

这个模块负责所有串口 DMA/IDLE 接收启动和回调分发。

建议管理这些串口：

| 串口     | 设备        |
| ------ | --------- |
| USART1 | printf 调试 |
| USART2 | WIT 陀螺仪   |
| USART3 | 张大头步进电机   |
| UART4  | 总线舵机      |
| UART5  | OPS       |
| USART6 | Jetson    |

`bsp_uart.h`：

```c
#ifndef __BSP_UART_H
#define __BSP_UART_H

#include "main.h"
#include "usart.h"

void BSP_UART_StartReceive(void);
void BSP_UART_RestartReceive(UART_HandleTypeDef *huart);

#endif
```

`bsp_uart.c` 应该做：

```text
1. 定义各串口 RX buffer
2. 启动 HAL_UARTEx_ReceiveToIdle_DMA()
3. 在 HAL_UARTEx_RxEventCallback() 中按串口分发数据
4. 在 HAL_UART_ErrorCallback() 中重启接收
```

注意：中断回调里不要解析复杂协议，只复制数据、置标志。

---

## 3. `zdt_stepper.c/h`

你已经完成张大头相关代码，这个模块继续保留。

它应该只负责：

```text
1. 张大头协议字节封装
2. 发送使能、失能、速度、位置、停止、同步启动命令
3. 解析电机返回 ACK
4. 解析速度、位置、状态返回
```

不要让它直接管“小车怎么运动”。
它是“电机驱动器协议层”，不是“底盘控制层”。

应该提供接口：

```c
void ZDT_Enable(uint8_t id);
void ZDT_Disable(uint8_t id);
void ZDT_SetSpeed(uint8_t id, uint8_t dir, uint16_t rpm, uint8_t acc, uint8_t sync);
void ZDT_SetPosition(uint8_t id, uint8_t dir, uint16_t rpm, uint8_t acc, uint32_t pulse, uint8_t mode, uint8_t sync);
void ZDT_Stop(uint8_t id);
void ZDT_SyncStart(void);
void ZDT_OnRxData(uint8_t *data, uint16_t len);
```

你之前整理的协议中，张大头控制命令是二进制字节帧，不是普通 ASCII 字符串；继续保持这种封装方式。

---

## 4. `app_chassis.c/h`

这个是后面最重要的应用层模块。

它负责把“车体运动”转换成“电机命令”。

例如：

```c
void Chassis_Init(void);
void Chassis_Stop(void);
void Chassis_EmergencyStop(void);
void Chassis_SetMotorRPM(int16_t lf, int16_t rf, int16_t lr, int16_t rr);
void Chassis_SetVelocity(float vx, float vy, float wz);
void Chassis_Task_10ms(void);
```

第一阶段你可以先只写：

```c
void Chassis_SetMotorRPM(int16_t lf, int16_t rf, int16_t lr, int16_t rr)
{
    // 根据正负号决定方向
    // 调用 ZDT_SetSpeed()
    // 最后调用 ZDT_SyncStart()
}
```

也就是说：

```text
app_chassis 调 ZDT
Jetson 后期只调 app_chassis
```

不要让 Jetson 协议层直接调用 `ZDT_SetSpeed()`，否则以后换电机协议会很麻烦。

void Servo_Init(void);

## 7. `ops_sensor.c/h`

OPS 定位传感器模块。

职责：

```text
1. 接收 UART5 数据
2. 解析 28 字节位姿帧
3. 维护最近一次定位数据
4. 给底盘或其它本地模块查询使用
```

接口：

```c

{
    float zangle_deg;
    float xangle_deg;
    float yangle_deg;
    float pos_x_mm;
    float pos_y_mm;
    float w_z_dps;
    uint32_t updated_tick;
    uint32_t frame_count;
    uint8_t valid;
} OPS_Pose_t;

void OPS_Init(UART_HandleTypeDef *huart);
void OPS_OnByteReceived(void);
void OPS_Poll(void);
void OPS_GetPose(OPS_Pose_t *pose);
```

## 8. `jetson_protocol.c/h`

Jetson 和 STM32 的通信协议模块。

职责：

```text
1. 接收 Jetson 发来的命令帧
2. 解析命令
3. 调用 app_chassis / servo / safety
4. 周期性向 Jetson 上报状态
```

建议先定义这些命令：

| 命令               | 方向             | 作用          |
| ---------------- | -------------- | ----------- |
| `PING`           | Jetson → STM32 | 通信测试        |
| `HEARTBEAT`      | Jetson → STM32 | 心跳          |
| `SET_MOTOR_RPM`  | Jetson → STM32 | 直接设置四轮 RPM  |
| `SET_VEL`        | Jetson → STM32 | 设置 vx/vy/wz |
| `SET_SERVO_POS`  | Jetson → STM32 | 设置舵机位置      |
| `EMERGENCY_STOP` | Jetson → STM32 | 急停          |
| `STATUS`         | STM32 → Jetson | 状态上报        |
| `ODOM`           | STM32 → Jetson | 里程计         |
| `IMU`            | STM32 → Jetson | 姿态数据        |

不要上来就写复杂 ROS2。先用 Python 串口脚本和 STM32 跑通协议。

## 9. `app_scheduler.c/h`

裸机任务调度器。

你现在没有必要马上上 FreeRTOS，先用定时器 tick + 主循环调度。

建议任务周期：

| 任务            |           周期 |
| ------------- | -----------: |
| LED 心跳        |        500ms |
| Jetson 心跳超时检测 |         10ms |
| 电机状态轮询        |         50ms |
| IMU 数据检查      |         10ms |
| OPS 数据检查      |         10ms |
| 状态上报给 Jetson  | 50ms / 100ms |
| 舵机状态查询        |        100ms |

接口：

```c
void Scheduler_Init(void);
void Scheduler_Run(void);

void Scheduler_1msHook(void);
```

`TIM6` 中断里只做：

```c
Scheduler_1msHook();
```

主循环里做：

```c
while (1)
{
    Scheduler_Run();
}
```

## 10. `app_safety.c/h`

安全保护模块，必须有。

职责：

```text
1. Jetson 通信超时急停
2. 电机异常状态急停
3. 急停按键处理
4. 上电默认不使能电机
5. 错误状态锁定
```

接口：

```c
void Safety_Init(void);
void Safety_Task_10ms(void);
void Safety_EmergencyStop(void);
void Safety_SetJetsonHeartbeatTime(uint32_t tick);
uint8_t Safety_IsEmergency(void);
```

关键逻辑：

```text
如果 300ms 没收到 Jetson 心跳：
    立即 ZDT_Stop(0x00)
```

这件事不要放在 Jetson 协议文件里，应该放在安全模块里。

# 四、主循环应该长什么样

`main.c` 不应该写业务细节，只负责初始化和调度。

建议结构：

```c
int main(void)
{
    HAL_Init();
    SystemClock_Config();

    MX_GPIO_Init();
    MX_DMA_Init();
    MX_USART1_UART_Init();
    MX_USART2_UART_Init();
    MX_USART3_UART_Init();
    MX_UART4_Init();
    MX_UART5_Init();
    MX_USART6_UART_Init();
    MX_TIM6_Init();

    BSP_LED_Init();
    BSP_UART_StartReceive();

    ZDT_Init();
    Chassis_Init();
    Safety_Init();
    Scheduler_Init();

    HAL_TIM_Base_Start_IT(&htim6);

    while (1)
    {
        Scheduler_Run();
    }
}
```

注意：
`MX_DMA_Init()` 要在串口初始化前面，这是多串口 DMA 接收的基础要求。

# 八、当前阶段最小可运行架构

你现在最小目标不是“接完所有设备”，而是做到：

```text
USART1：打印日志
USART3：控制张大头步进电机
TIM6：1ms调度

app_safety：支持急停
```

最小闭环：

```text
main
 → Scheduler_Run
 → Chassis_SetMotorRPM
 → ZDT_SetSpeed
 → USART3发送
 → 电机运动
 → USART3接收ACK
 → ZDT_OnRxData更新状态
```

这个跑通后，再接舵机、Jetson、IMU、OPS。否则你会在多外设联调时失去定位问题的能力。
