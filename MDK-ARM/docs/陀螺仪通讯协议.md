# 陀螺仪解析文档

## 1. 目标范围

当前业务只保留三类三轴量：

- 加速度
- 角速度
- 角度

对应 WIT 标准串口协议帧如下：

- `0x51`：三轴加速度
- `0x52`：三轴角速度
- `0x53`：三轴角度

除上述内容外，其余帧型、配置命令、磁场、气压、四元数等内容均不纳入本轮解析范围。

## 2. 协议结论

根据 HWT905 手册 `5 Communication Protocol`：

- 每帧固定 11 字节，帧头为 `0x55`
- 三轴原始值均按小端有符号 `int16_t` 解析
- `0x51` 量程按 `16g` 换算
- `0x52` 量程按 `2000 deg/s` 换算
- `0x53` 量程按 `180 deg` 换算

需要特别注意：

- 当前工程对外直接暴露角速度，不再做“角加速度差分”
- `0x53` 帧尾的 `Byte8/Byte9` 是版本号，不是温度
- 当前工程走的是 `USART2` 标准串口解析，不是蓝牙组合包解析

## 3. 单帧格式

```text
Byte0   0x55
Byte1   FrameType
Byte2   Data1_L
Byte3   Data1_H
Byte4   Data2_L
Byte5   Data2_H
Byte6   Data3_L
Byte7   Data3_H
Byte8   Data4_L
Byte9   Data4_H
Byte10  CheckSum
```

校验规则：

```text
CheckSum = (Byte0 + Byte1 + ... + Byte9) 的低 8 位
```

当前只关心 `0x51`、`0x52`、`0x53`，其余类型直接忽略。

## 4. 数值换算

### 4.1 加速度 `0x51`

```text
ax = raw_ax / 32768 * 16
ay = raw_ay / 32768 * 16
az = raw_az / 32768 * 16
```

单位：`g`

### 4.2 角速度 `0x52`

```text
wx = raw_wx / 32768 * 2000
wy = raw_wy / 32768 * 2000
wz = raw_wz / 32768 * 2000
```

单位：`deg/s`

### 4.3 角度 `0x53`

```text
roll  = raw_roll  / 32768 * 180
pitch = raw_pitch / 32768 * 180
yaw   = raw_yaw   / 32768 * 180
```

单位：`deg`

## 5. 缓存设计

模块对外提供统一结构体：

```c
typedef struct
{
  WIT_Vector3f_t accel_g;
  WIT_Vector3f_t gyro_dps;
  WIT_Vector3f_t angle_deg;
} WIT_Data_t;
```

每个向量都包含：

- `x / y / z`
- `updated_tick`
- `valid`

其中：

- `accel_g.x/y/z` 对应 X/Y/Z 三轴加速度
- `gyro_dps.x/y/z` 对应 X/Y/Z 三轴角速度
- `angle_deg.x/y/z` 对应 X/Y/Z 三轴角度

## 6. 解析流程

- 串口 DMA 回调拿到一段原始字节流
- 与上一次未解析完的残留字节拼接
- 从头扫描 `0x55`
- 长度足够 11 字节时做校验
- 校验通过后按 `0x51/0x52/0x53` 更新对应缓存
- 长度不足 11 字节时保留残留，等待下一次回调继续拼接
