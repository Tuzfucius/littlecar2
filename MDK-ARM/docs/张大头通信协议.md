# 张大头闭环步进电机通信协议封装文档

## 1. 使用场景

本项目中，STM32F407 作为下位机，通过一个串口总线控制多个张大头闭环步进电机。每个电机拥有独立 ID，STM32 按 ID 发送指令，电机根据自身 ID 判断是否执行。

推荐在 STM32F407 上分配：

```text
USART3 → 步进电机扩展板
PB10   → USART3_TX
PB11   → USART3_RX
GND    → GND
```

如果你的扩展板是 TTL 串口，则 TX/RX 交叉连接；如果是 RS485 总线，则 STM32 USART3 需要经过 MAX485/SP3485 转换后再接 A/B 线。

---

## 2. 串口基础设置

官方说明中，上位机/串口助手需要勾选 **16进制发送、16进制显示**，取消“发送新行”，默认波特率为 **115200**。([CSDN][1])

推荐 STM32 串口配置：

```text
BaudRate: 115200
DataBits: 8
StopBits: 1
Parity: None
Mode: TX/RX
```

也就是：

```text
115200, 8N1
```

注意：协议命令是**十六进制字节流**，不是 ASCII 字符串。

错误写法：

```c
HAL_UART_Transmit(&huart3, (uint8_t *)"01 F6 00 00 64 0A 00 6B", 23, 100);
```

正确写法：

```c
uint8_t cmd[] = {0x01, 0xF6, 0x00, 0x00, 0x64, 0x0A, 0x00, 0x6B};
HAL_UART_Transmit(&huart3, cmd, sizeof(cmd), 100);
```

---

## 3. 电机 ID 规划

官方协议中，`ID_Addr` 默认是 `1`，可设置范围为 `1~255`，`0` 是广播地址；用广播地址发送命令时，所有电机会执行，但只有地址为 `1` 的电机会回复。([CSDN][1])

建议你的小车这样分配：

### 两轮差速小车

| 电机 | ID     |
| -- | ------ |
| 左轮 | `0x01` |
| 右轮 | `0x02` |

### 四轮 / 麦轮小车

| 电机位置   | ID     |
| ------ | ------ |
| 左前轮 LF | `0x01` |
| 右前轮 RF | `0x02` |
| 左后轮 LR | `0x03` |
| 右后轮 RR | `0x04` |

不要一开始用复杂 ID。第一版用 `1,2,3,4` 最清晰。

---

## 4. 总体帧格式

Emm_V5.0 通信控制命令的发送和接收格式为：

```text
地址 + 功能码 + 指令数据 + 校验字节
```

官方文档中字段名为：

```text
ID_Addr + Func_Code + Data + CheckSum
```

校验方式支持固定 `0x6B`、XOR 校验、CRC-8 校验三种，默认是 `0x6B`。([CSDN][1])

### 4.1 通用帧结构

```text
Byte0      Byte1        Byte2...ByteN-2      ByteN-1
ID地址     功能码        指令数据             校验字节
```

### 4.2 字段说明

| 字段          |     长度 | 含义                          |
| ----------- | -----: | --------------------------- |
| `ID_Addr`   | 1 byte | 电机地址，`0x01~0xFF`；`0x00` 为广播 |
| `Func_Code` | 1 byte | 功能码，例如 `0xF6` 速度模式          |
| `Data`      |     变长 | 具体命令参数                      |
| `CheckSum`  | 1 byte | 默认固定 `0x6B`                 |

### 4.3 多字节数据字节序

协议里的多字节数一般采用**高字节在前**。

例如速度 `1500 RPM`：

```text
1500(dec) = 0x05DC
发送为：05 DC
```

脉冲数 `32000`：

```text
32000(dec) = 0x00007D00
发送为：00 00 7D 00
```

---

## 5. 校验字节规则

## 5.1 固定校验 `0x6B`

第一版建议使用默认方式：

```text
CheckSum = 0x6B
```

即每一条命令最后一个字节固定为：

```text
6B
```

这是最适合你当前 STM32 调试阶段的方式。

---

## 5.2 XOR 校验

若驱动器菜单中把校验方式改成 XOR，则最后一个字节等于前面所有字节异或。官方示例说明，`0x42 = 0x01 ^ 0x06 ^ 0x45`。([CSDN][1])

示例：

```text
01 06 45 42
```

计算：

```c
0x01 ^ 0x06 ^ 0x45 = 0x42
```

---

## 5.3 CRC-8 校验

驱动器也支持 CRC-8，但当前不建议你用。因为你需要确认多项式、初值、反射规则，否则容易出现“看起来代码对了但电机不响应”的问题。

项目第一版统一使用：

```text
固定 0x6B 校验
```

# 6. 通用返回格式

大多数控制类命令返回：

```text
地址 + 功能码 + 命令状态 + 校验字节
```

常见状态码：

| 状态码    | 含义    |
| ------ | ----- |
| `0x02` | 命令正确  |
| `0xE2` | 条件不满足 |
| `0xEE` | 错误命令  |

例如速度模式中，正确返回：

```text
01 F6 02 6B
```

若条件不满足，例如堵转保护触发或电机未使能，则可能返回：

```text
01 F6 E2 6B
```

错误命令通常返回：

```text
01 00 EE 6B
```

速度、位置、使能等命令的官方说明中均给出了上述返回规则。([CSDN][1])

# 7. 核心控制命令

## 7.1 电机使能控制

### 命令格式

```text
地址 + F3 + AB + 使能状态 + 多机同步标志 + 6B
```

官方格式为：`地址 + 0xF3 + 0xAB + 使能状态 + 多机同步标志 + 校验字节`。([CSDN][1])

### 字段说明

| 字段   | 示例   | 含义                    |
| ---- | ---- | --------------------- |
| 地址   | `01` | 电机 ID                 |
| 功能码  | `F3` | 使能控制                  |
| 固定字节 | `AB` | 该命令固定参数               |
| 使能状态 | `01` | `01` 使能，`00` 失能       |
| 同步标志 | `00` | `00` 立即执行，`01` 等待同步启动 |
| 校验   | `6B` | 默认校验                  |

### 示例

使能 ID=1：

```text
01 F3 AB 01 00 6B
```

失能 ID=1：

```text
01 F3 AB 00 00 6B
```

返回：

```text
01 F3 02 6B
```

---

## 7.2 速度模式控制

### 命令格式

```text
地址 + F6 + 方向 + 速度高字节 + 速度低字节 + 加速度 + 多机同步标志 + 6B
```

官方格式为：`地址 + 0xF6 + 方向 + 速度 + 加速度 + 多机同步标志 + 校验字节`。([CSDN][1])

### 字段说明

| 字段   | 长度 | 含义                   |
| ---- | -: | -------------------- |
| 地址   |  1 | 电机 ID                |
| `F6` |  1 | 速度模式功能码              |
| 方向   |  1 | `00` = CW，`01` = CCW |
| 速度   |  2 | 单位 RPM，高字节在前         |
| 加速度  |  1 | 加速度档位                |
| 同步标志 |  1 | `00` 立即执行，`01` 等待同步  |
| 校验   |  1 | 默认 `6B`              |

### 官方示例解析

```text
01 F6 01 05 DC 0A 00 6B
```

含义：

| 字节      | 含义                  |
| ------- | ------------------- |
| `01`    | ID=1                |
| `F6`    | 速度模式                |
| `01`    | CCW 方向              |
| `05 DC` | `0x05DC = 1500 RPM` |
| `0A`    | 加速度档位 10            |
| `00`    | 不启用多机同步             |
| `6B`    | 校验                  |

官方文档说明 `01` 表示 CCW，`00` 表示 CW，`05 DC` 为 1500 RPM，`0A` 为加速度档位 10。([CSDN][1])

### 低速测试示例

100 RPM 正转：

```text
01 F6 00 00 64 0A 00 6B
```

100 RPM 反转：

```text
01 F6 01 00 64 0A 00 6B
```

---

## 7.3 位置模式控制

### 命令格式

```text
地址 + FD + 方向 + 速度 + 加速度 + 脉冲数 + 相对/绝对模式标志 + 多机同步标志 + 6B
```

官方格式为：`地址 + 0xFD + 方向 + 速度 + 加速度 + 脉冲数 + 相对/绝对模式标志 + 多机同步标志 + 校验字节`。([CSDN][1])

### 字段说明

| 字段   | 长度 | 含义                   |
| ---- | -: | -------------------- |
| 地址   |  1 | 电机 ID                |
| `FD` |  1 | 位置模式功能码              |
| 方向   |  1 | `00` = CW，`01` = CCW |
| 速度   |  2 | RPM，高字节在前            |
| 加速度  |  1 | 加速度档位                |
| 脉冲数  |  4 | 目标脉冲数，高字节在前          |
| 位置模式 |  1 | `00` 相对位置，`01` 绝对位置  |
| 同步标志 |  1 | `00` 立即执行，`01` 等待同步  |
| 校验   |  1 | 默认 `6B`              |

### 官方示例解析

```text
01 FD 01 05 DC 00 00 00 7D 00 00 00 6B
```

含义：

| 字节            | 含义                       |
| ------------- | ------------------------ |
| `01`          | ID=1                     |
| `FD`          | 位置模式                     |
| `01`          | CCW                      |
| `05 DC`       | 1500 RPM                 |
| `00`          | 加速度档位 0                  |
| `00 00 7D 00` | `0x00007D00 = 32000` 个脉冲 |
| `00`          | 相对位置                     |
| `00`          | 不启用同步                    |
| `6B`          | 校验                       |

官方说明在 16 细分下，发送 3200 个脉冲电机旋转一圈，所以 32000 个脉冲表示 10 圈。([CSDN][1])

---

## 7.4 立即停止

### 命令格式

```text
地址 + FE + 98 + 多机同步标志 + 6B
```

官方格式为：`地址 + 0xFE + 0x98 + 多机同步标志 + 校验字节`，可用于速度模式和位置模式下的紧急刹车。([CSDN][1])

### 示例

停止 ID=1：

```text
01 FE 98 00 6B
```

广播停止所有电机：

```text
00 FE 98 00 6B
```

返回：

```text
01 FE 02 6B
```

---

## 7.5 多机同步运动

### 用途

如果你有多个电机，逐个发速度/位置命令会有微小时间差。多机同步运动的做法是：

```text
1. 给每个电机发送运动命令，并将“多机同步标志”设为 01
2. 所有电机收到命令后暂不运动
3. 最后发送同步启动命令
4. 多个电机同时开始运动
```

官方多机同步示例中，先给 1 号和 2 号电机发送同步标志为 `01` 的位置命令，电机暂不运动；最后发送 `00 FF 66 6B`，两个电机开始同步运动。([CSDN][1])

### 同步启动命令

```text
地址 + FF + 66 + 6B
```

建议用广播地址：

```text
00 FF 66 6B
```

## 8.4 读取实时转速

命令：

```text
01 35 6B
```

返回：

```text
地址 + 35 + 符号 + 转速高字节 + 转速低字节 + 6B
```

字段：

| 字段      | 含义  |
| ------- | --- |
| 符号 `00` | 正数  |
| 符号 `01` | 负数  |
| 转速      | RPM |

官方示例中，`01 35 01 05 DC 6B` 表示实时转速为 `-1500 RPM`。([CSDN][1])

## 8.5 读取实时位置

命令：

```text
01 36 6B
```

返回：

```text
地址 + 36 + 符号 + 位置4字节 + 6B
```

官方说明中，实时位置值 `0~65535` 表示一圈，可转换为角度：

```text
角度 = 位置值 * 360 / 65536
```

如果符号字节为 `01`，则为负数。([CSDN][1])

---

## 8.6 读取电机状态标志位

命令：

```text
01 3A 6B
```

返回：

```text
地址 + 3A + 状态标志 + 6B
```

状态标志按 bit 判断。官方示例中，返回 `01 3A 03 6B` 时，可通过位运算判断使能状态、到位状态、堵转状态和堵转保护状态。([CSDN][1])

### 状态位定义

| bit  | 掩码     | 含义     |
| ---- | ------ | ------ |
| bit0 | `0x01` | 电机使能状态 |
| bit1 | `0x02` | 电机到位状态 |
| bit2 | `0x04` | 电机堵转状态 |
| bit3 | `0x08` | 堵转保护状态 |

解析示例：

```c
uint8_t status = 0x03;

bool enabled = status & 0x01;
bool reached = status & 0x02;
bool stalled = status & 0x04;
bool stall_protect = status & 0x08;
```

## 8.7 读取系统状态参数

命令：

```text
01 43 7A 6B
```

返回：

```text
地址 + 43 + 系统状态参数 + 6B
```

官方系统状态参数中包含总线电压、相电流、校准后编码器值、电机目标位置、实时转速、实时位置、位置误差、就绪状态标志、电机状态标志等。([CSDN][1])

这个命令适合做周期性状态查询，但数据较长。初期调试建议先分别读取 `0x35` 实时转速、`0x36` 实时位置、`0x3A` 状态标志。

# 9. 修改 ID 命令

## 9.1 命令格式

```text
地址 + AE + 4B + 是否存储标志 + 新ID + 6B
```

官方格式为：`地址 + 0xAE + 0x4B + 是否存储标志 + ID地址 + 校验字节`。示例 `01 AE 4B 01 10 6B` 表示把 ID 修改为 `0x10 = 16` 并存储。([CSDN][1])

### 字段说明

| 字段      | 含义                    |
| ------- | --------------------- |
| 地址      | 当前电机 ID               |
| `AE 4B` | 修改 ID 固定命令            |
| 是否存储    | `00` 不保存，断电丢失；`01` 保存 |
| 新 ID    | 目标地址                  |
| `6B`    | 校验                    |

### 示例

把当前 ID=1 的电机改成 ID=2，并保存：

```text
01 AE 4B 01 02 6B
```

把当前 ID=1 的电机改成 ID=3，并保存：

```text
01 AE 4B 01 03 6B
```

## 9.2 修改 ID 的正确流程

必须一次只接一个电机：

```text
1. 总线上只接一个电机
2. 确认当前 ID，通常默认是 1
3. 发送修改 ID 命令
4. 等待返回成功
5. 断电重启
6. 用新 ID 发送读取版本/使能命令验证
7. 贴标签，记录该电机 ID
```

不要把多个默认 ID=1 的电机同时挂在总线上修改，否则可能所有电机都被改成同一个 ID。

# 10. STM32 封装规则

## 10.1 基础封装原则

你在 STM32 中不要到处手写数组，应该封装统一接口：

```c
ZDT_Enable(id);
ZDT_Disable(id);
ZDT_SetSpeed(id, dir, rpm, acc, sync);
ZDT_SetPosition(id, dir, rpm, acc, pulses, mode, sync);
ZDT_Stop(id);
ZDT_SyncStart();
ZDT_ReadSpeed(id);
ZDT_ReadPosition(id);
ZDT_ReadStatus(id);
ZDT_SetID(old_id, new_id);
```

这样后期 Jetson 下发底盘速度时，只需要调用这些接口，不用关心底层字节。

---

## 10.2 类型定义

```c
typedef enum
{
    ZDT_DIR_CW  = 0x00,
    ZDT_DIR_CCW = 0x01
} ZDT_Dir_t;

typedef enum
{
    ZDT_SYNC_DISABLE = 0x00,
    ZDT_SYNC_ENABLE  = 0x01
} ZDT_Sync_t;

typedef enum
{
    ZDT_POS_RELATIVE = 0x00,
    ZDT_POS_ABSOLUTE = 0x01
} ZDT_PosMode_t;
```

---

## 10.3 基础发送函数

```c
#include "usart.h"
#include <stdint.h>
#include <stdbool.h>

#define ZDT_CHECKSUM_FIXED  0x6B

static void ZDT_SendBytes(uint8_t *data, uint16_t len)
{
    HAL_UART_Transmit(&huart3, data, len, 100);
}

static void ZDT_PutU16BE(uint8_t *buf, uint16_t value)
{
    buf[0] = (uint8_t)(value >> 8);
    buf[1] = (uint8_t)(value & 0xFF);
}

static void ZDT_PutU32BE(uint8_t *buf, uint32_t value)
{
    buf[0] = (uint8_t)(value >> 24);
    buf[1] = (uint8_t)(value >> 16);
    buf[2] = (uint8_t)(value >> 8);
    buf[3] = (uint8_t)(value & 0xFF);
}
```

---

## 10.4 使能 / 失能

```c
void ZDT_Enable(uint8_t id)
{
    uint8_t cmd[] = {
        id,
        0xF3,
        0xAB,
        0x01,
        0x00,
        ZDT_CHECKSUM_FIXED
    };

    ZDT_SendBytes(cmd, sizeof(cmd));
}

void ZDT_Disable(uint8_t id)
{
    uint8_t cmd[] = {
        id,
        0xF3,
        0xAB,
        0x00,
        0x00,
        ZDT_CHECKSUM_FIXED
    };

    ZDT_SendBytes(cmd, sizeof(cmd));
}
```

## 10.5 速度模式

```c
void ZDT_SetSpeed(uint8_t id, ZDT_Dir_t dir, uint16_t rpm, uint8_t acc, ZDT_Sync_t sync)
{
    uint8_t cmd[8];

    cmd[0] = id;
    cmd[1] = 0xF6;
    cmd[2] = (uint8_t)dir;
    ZDT_PutU16BE(&cmd[3], rpm);
    cmd[5] = acc;
    cmd[6] = (uint8_t)sync;
    cmd[7] = ZDT_CHECKSUM_FIXED;

    ZDT_SendBytes(cmd, sizeof(cmd));
}
```

调用示例：

```c
ZDT_Enable(1);
HAL_Delay(100);

ZDT_SetSpeed(1, ZDT_DIR_CW, 100, 10, ZDT_SYNC_DISABLE);
HAL_Delay(2000);

ZDT_Stop(1);
```

## 10.6 位置模式

```c
void ZDT_SetPosition(uint8_t id,
                     ZDT_Dir_t dir,
                     uint16_t rpm,
                     uint8_t acc,
                     uint32_t pulses,
                     ZDT_PosMode_t mode,
                     ZDT_Sync_t sync)
{
    uint8_t cmd[13];

    cmd[0] = id;
    cmd[1] = 0xFD;
    cmd[2] = (uint8_t)dir;
    ZDT_PutU16BE(&cmd[3], rpm);
    cmd[5] = acc;
    ZDT_PutU32BE(&cmd[6], pulses);
    cmd[10] = (uint8_t)mode;
    cmd[11] = (uint8_t)sync;
    cmd[12] = ZDT_CHECKSUM_FIXED;

    ZDT_SendBytes(cmd, sizeof(cmd));
}
```

调用示例：ID=1，100RPM，相对运动 3200 脉冲。

                10,
                3200,

                ZDT_SYNC_DISABLE);

# 11. 返回数据解析规则

## 11.1 控制类返回解析

以速度模式返回为例：

```text
01 F6 02 6B
```

解析规则：

| 字节   | 含义      |
| ---- | ------- |
| `01` | 返回电机 ID |
| `F6` | 对应功能码   |
| `02` | 命令状态，成功 |
| `6B` | 校验      |

推荐写一个通用判断函数：

```c
bool ZDT_IsAckOK(uint8_t *rx, uint16_t len, uint8_t expected_id, uint8_t expected_func)
{
    if (len < 4) return false;
    if (rx[0] != expected_id) return false;
    if (rx[1] != expected_func) return false;
    if (rx[len - 1] != ZDT_CHECKSUM_FIXED) return false;

    return rx[2] == 0x02;
}
```

如果使用广播地址 `0x00`，不能按常规方式等待所有电机回复。协议说明广播时只有 ID=1 的电机会回复，因此广播指令更适合“发出动作”，不适合“逐个确认”。([CSDN][1])

---

## 11.2 读取实时速度解析

返回示例：

```text
01 35 01 05 DC 6B
```

解析：

```c
int16_t ZDT_ParseSpeed(uint8_t *rx)
{
    uint8_t sign = rx[2];
    uint16_t raw = ((uint16_t)rx[3] << 8) | rx[4];

    if (sign == 0x01)
        return -(int16_t)raw;
    else
        return (int16_t)raw;
}
```

---

## 11.3 读取实时位置解析

返回示例：

```text
01 36 01 00 01 00 00 6B
```

解析：

```c
int32_t ZDT_ParsePositionRaw(uint8_t *rx)
{
    uint8_t sign = rx[2];

    uint32_t raw =
        ((uint32_t)rx[3] << 24) |
        ((uint32_t)rx[4] << 16) |
        ((uint32_t)rx[5] << 8)  |
        ((uint32_t)rx[6]);

    if (sign == 0x01)
        return -(int32_t)raw;
    else
        return (int32_t)raw;
}

float ZDT_PositionRawToDegree(int32_t raw)
{
    return ((float)raw * 360.0f) / 65536.0f;
}
```

# 13. 小车控制层封装建议

底层协议函数不要直接被 Jetson 调用。建议在 STM32 中再封一层底盘接口。

```c
typedef enum
{
    MOTOR_LF = 1,
    MOTOR_RF = 2,
    MOTOR_LR = 3,
    MOTOR_RR = 4
} ChassisMotor_t;
```

例如：

```c
void Chassis_SetMotorRPM(ChassisMotor_t motor, int16_t rpm)
{
    ZDT_Dir_t dir;
    uint16_t abs_rpm;

    if (rpm >= 0)
    {
        dir = ZDT_DIR_CW;
        abs_rpm = rpm;
    }
    else
    {
        dir = ZDT_DIR_CCW;
        abs_rpm = -rpm;
    }

    ZDT_SetSpeed((uint8_t)motor, dir, abs_rpm, 10, ZDT_SYNC_ENABLE);
}
```

四轮同步：

```c
void Chassis_SetFourMotorRPM(int16_t lf, int16_t rf, int16_t lr, int16_t rr)
{
    Chassis_SetMotorRPM(MOTOR_LF, lf);
    Chassis_SetMotorRPM(MOTOR_RF, rf);
    Chassis_SetMotorRPM(MOTOR_LR, lr);
    Chassis_SetMotorRPM(MOTOR_RR, rr);

    ZDT_SyncStart();
}
```

# 14. 注意事项

## 14.1 不要混淆“字符串”和“字节”

你以前说“发字符串控制”，实际很多串口工具里你看到的是：

```text
01 F6 00 00 64 0A 00 6B
```

但这不是普通文本字符串，而是十六进制字节。STM32 里必须发：

```c
{0x01, 0xF6, 0x00, 0x00, 0x64, 0x0A, 0x00, 0x6B}
```

---

## 14.2 先调单电机，再调多电机

推荐顺序：

```text
1. 只接一个电机
2. 读版本
3. 使能
4. 低速运动
5. 停止
6. 修改 ID
7. 贴标签
8. 接入第二个电机
9. 测试多机同步
```

---

## 14.3 广播地址慎用

广播地址 `0x00` 很适合：

```text
急停
同步启动
统一速度命令
```

但不适合：

```text
读取状态
修改 ID
参数配置
```

因为广播时并不是所有设备都会回复，且修改 ID 时容易把多个设备改成同一个 ID。

---

## 14.4 下位机必须保留急停接口

STM32 中应封装：

```c
void Chassis_EmergencyStop(void)
{
    ZDT_Stop(0x00);
}
```

后面 Jetson 通信超时、遥控器急停、串口异常，都调用这个函数。

# 15. 推荐文件结构

你的 STM32 工程可以这样组织：

```text
BSP/
  zdt_stepper/
    zdt_stepper.h
    zdt_stepper.c

App/
  chassis/
    chassis.h
    chassis.c
```

`zdt_stepper` 只负责协议封装。

`chassis` 负责小车运动逻辑。

不要把底盘运动学、Jetson 通信、协议字节全部写在 `main.c` 里。后面会很难维护。
