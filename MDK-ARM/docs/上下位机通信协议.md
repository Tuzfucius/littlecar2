# STM32F407 上下位机通信协议文档

## 1. 协议目标

本协议用于：

```text
Jetson Orin Nano 上位机  ↔  STM32F407 下位机
```

主要目标：

```text
1. Jetson 发送高级控制命令
2. STM32 执行实时运动控制
3. STM32 控制张大头步进电机、总线舵机、机械臂、夹爪
4. STM32 采集 OPS、IMU、电机状态等数据
5. STM32 周期性向 Jetson 上报状态
6. 支持急停、心跳超时保护、错误反馈
7. 协议具备后续扩展能力
```

你上传的原始草案采用了二进制帧、双帧头 `0x5A 0xA5`、长度字段、指令码、校验和、帧尾，并将指令分为系统安全、底盘、机械臂、状态上报等号段，这是正确方向；但为了 STM32 实现更稳定，这里改为**受控变长帧 + 核心命令定长 Payload + CRC16 校验**。

---

# 2. 总体设计结论

本协议不是纯定长帧，也不是随意不定长帧。

采用：

```text
外层：受控变长帧
内层：核心控制命令定长 Payload
扩展：配置类、批量类命令允许变长 Payload
```

也就是：

```text
实时控制命令：定长
配置扩展命令：变长
整体帧：带 Length + CRC16
```

这样做的原因：

```text
1. STM32 解析稳定
2. 支持粘包、断包恢复
3. 支持后续功能扩展
4. 不浪费太多通信带宽
5. 不把张大头电机底层协议暴露给 Jetson
```

# 3. 通信链路建议

## 3.1 物理串口

建议上下位机主通信使用：

```text
USART6
```

引脚分配：

| Jetson | STM32F407       |
| ------ | --------------- |
| TX     | PC7 / USART6_RX |
| RX     | PC6 / USART6_TX |
| GND    | GND             |

通信参数：

```text
BaudRate: 初期 115200，稳定后可提高到 460800 / 921600
DataBits: 8
StopBits: 1
Parity: None
FlowControl: None
```

即：

```text
115200, 8N1
```

## 3.2 STM32 接收方式

STM32 端建议：

```text
USART6_RX：DMA + IDLE 空闲中断
```

不要使用阻塞接收：

```c
HAL_UART_Receive(...)
```

推荐使用：

```c
HAL_UARTEx_ReceiveToIdle_DMA(...)
```

中断回调只负责接收和缓存，不直接执行底盘或机械臂动作。

# 4. 协议帧格式

## 4.1 最终帧结构

取消帧尾 `0xFF`，只依赖：

```text
双帧头 + Length + CRC16
```

最终格式：

```text
Header1 Header2 Version FrameType CmdSet CmdID Flags Seq_L Seq_H Length Payload CRC_L CRC_H
```

字段表：

|    偏移 | 字段        | 长度 | 类型      | 说明               |
| ----: | --------- | -: | ------- | ---------------- |
|     0 | Header1   |  1 | uint8_t | 固定 `0x5A`        |
|     1 | Header2   |  1 | uint8_t | 固定 `0xA5`        |
|     2 | Version   |  1 | uint8_t | 协议版本，当前 `0x01`   |
|     3 | FrameType |  1 | uint8_t | 帧类型              |
|     4 | CmdSet    |  1 | uint8_t | 命令组              |
|     5 | CmdID     |  1 | uint8_t | 命令 ID            |
|     6 | Flags     |  1 | uint8_t | 标志位              |
|     7 | Seq_L     |  1 | uint8_t | 序号低字节            |
|     8 | Seq_H     |  1 | uint8_t | 序号高字节            |
|     9 | Length    |  1 | uint8_t | Payload 长度，0~255 |
| 10... | Payload   |  N | bytes   | 数据区              |
|  10+N | CRC_L     |  1 | uint8_t | CRC16 低字节        |
|  11+N | CRC_H     |  1 | uint8_t | CRC16 高字节        |

基础头长度：

```c
#define PROTO_BASE_HEAD_LEN  10
```

整帧长度：

```c
frame_total_len = 10 + Length + 2;
```

---

## 4.2 为什么不用帧尾

二进制数据区里可能自然出现：

```text
0xFF
0x5A
0xA5
```

如果依赖帧尾，容易误判。

更稳的解析方式是：

```text
1. 用 0x5A 0xA5 找帧起点
2. 用 Length 确定 Payload 长度
3. 用 CRC16 判断整帧是否有效
4. CRC 错误则丢弃并重新寻找下一组帧头
```

# 5. CRC16 校验

## 5.1 CRC 算法

采用：

```text
CRC16-Modbus
初值：0xFFFF
多项式：0xA001
输出：低字节在前
```

## 5.2 CRC 计算范围

CRC16 计算范围：

```text
Version 到 Payload 最后一个字节
```

不包含：

```text
Header1
Header2
CRC_L
CRC_H
```

也就是：

```c
crc = CRC16_Modbus(&frame[2], 8 + payload_len);
```

解释：

```text
frame[2] = Version
从 Version 到 Length 共 8 字节
再加 Payload 长度 N
```

# 6. 字节序与数据类型

## 6.1 字节序

统一使用：

```text
Little-Endian 小端
```

示例：

```text
uint16_t 300  = 0x012C      → 2C 01
int32_t 1000  = 0x000003E8  → E8 03 00 00
int32_t 9000  = 0x00002328  → 28 23 00 00
```

## 6.2 不建议直接传 float

STM32 与 Jetson 都能处理 float，但正式协议不建议把 float 作为主协议字段。

推荐使用定点整数：

| 物理量        | 类型                  | 单位      |
| ---------- | ------------------- | ------- |
| x / y 位置   | int32_t             | mm      |
| yaw 角度     | int32_t             | 0.01°   |
| vx / vy 速度 | int16_t             | mm/s    |
| wz 角速度     | int16_t             | 0.01°/s |
| 电机转速       | int16_t             | RPM     |
| 舵机位置       | uint16_t            | 舵机原始值   |
| 时间         | uint16_t / uint32_t | ms      |
| 电压         | uint16_t            | mV      |
| 电流         | uint16_t            | mA      |

示例：

```text
前进 1m      → dx_mm = 1000
旋转 90°     → dyaw_cdeg = 9000
速度 0.3m/s  → vx_mm_s = 300
```

# 7. FrameType 帧类型

```c
typedef enum
{
    FRAME_CMD    = 0x01,
    FRAME_ACK    = 0x02,
    FRAME_EVENT  = 0x03,
    FRAME_STATUS = 0x04,
    FRAME_RESP   = 0x05,
    FRAME_ERROR  = 0x7F
} FrameType_t;
```

| FrameType | 名称     | 方向             | 含义          |
| --------: | ------ | -------------- | ----------- |
|    `0x01` | CMD    | Jetson → STM32 | 命令帧         |
|    `0x02` | ACK    | STM32 → Jetson | 命令接收确认      |
|    `0x03` | EVENT  | STM32 → Jetson | 动作完成 / 异常事件 |
|    `0x04` | STATUS | STM32 → Jetson | 周期状态上报      |
|    `0x05` | RESP   | STM32 → Jetson | 查询响应        |
|    `0x7F` | ERROR  | STM32 → Jetson | 协议错误或执行错误   |

重点：

```text
ACK 只表示“命令已收到并接受”
EVENT 才表示“动作完成或失败”
```

# 8. Flags 标志位

```c
#define FLAG_ACK_REQUIRED    0x01
#define FLAG_EVENT_REQUIRED  0x02
#define FLAG_HIGH_PRIORITY   0x04
#define FLAG_EMERGENCY       0x08
#define FLAG_NEED_RESP       0x10
#define FLAG_RESERVED_5      0x20
#define FLAG_RESERVED_6      0x40
#define FLAG_RESERVED_7      0x80
```

|    bit |     掩码 | 名称             | 含义              |
| -----: | -----: | -------------- | --------------- |
|   bit0 | `0x01` | ACK_REQUIRED   | 需要 STM32 返回 ACK |
|   bit1 | `0x02` | EVENT_REQUIRED | 动作完成后需要 EVENT   |
|   bit2 | `0x04` | HIGH_PRIORITY  | 高优先级命令          |
|   bit3 | `0x08` | EMERGENCY      | 急停/安全命令         |
|   bit4 | `0x10` | NEED_RESP      | 查询命令需要 RESP     |
| bit5~7 |      - | RESERVED       | 保留              |

常用组合：

| 场景                  |  Flags |
| ------------------- | -----: |
| 普通命令，需要 ACK         | `0x01` |
| 运动命令，需要 ACK + EVENT | `0x03` |
| 急停命令                | `0x0D` |
| 查询命令                | `0x11` |
| 状态上报                | `0x00` |

# 9. CmdSet 命令组

```c
typedef enum
{
    CMDSET_SYSTEM  = 0x01,
    CMDSET_SAFETY  = 0x02,
    CMDSET_CHASSIS = 0x03,
    CMDSET_ARM     = 0x04,
    CMDSET_SENSOR  = 0x05,
    CMDSET_CONFIG  = 0x06,
    CMDSET_TASK    = 0x07,
    CMDSET_DEBUG   = 0x7F
} CmdSet_t;
```

| CmdSet | 名称      | 用途            |
| -----: | ------- | ------------- |
| `0x01` | SYSTEM  | 心跳、模式、清错      |
| `0x02` | SAFETY  | 急停、安全停止       |
| `0x03` | CHASSIS | 底盘运动          |
| `0x04` | ARM     | 机械臂、夹爪、舵机     |
| `0x05` | SENSOR  | OPS、IMU、测距等   |
| `0x06` | CONFIG  | 点位、机械臂预设、参数配置 |
| `0x07` | TASK    | 任务序列          |
| `0x7F` | DEBUG   | 调试命令          |

采用 `CmdSet + CmdID`，而不是单独 `CmdID`，便于后续扩展。

### SYS_HEARTBEAT

用途：Jetson 心跳。

建议 Jetson 每：

```text
50ms 或 100ms
```

发送一次。

Payload 定长：

```text
Length = 5
```

| 偏移 | 字段             | 类型       | 长度 |
| -: | -------------- | -------- | -: |
|  0 | jetson_time_ms | uint32_t |  4 |
|  4 | control_mode   | uint8_t  |  1 |

`control_mode`：

|      值 | 模式          |
| -----: | ----------- |
| `0x00` | IDLE        |
| `0x01` | DEBUG       |
| `0x02` | MANUAL      |
| `0x03` | AUTO_NAV    |
| `0x04` | TASK_EXEC   |
| `0x05` | CALIBRATION |
| `0x06` | SAFE_STOP   |

安全规则：

```text
STM32 超过 300ms 没收到 HEARTBEAT：
1. 底盘立即停止
2. 机械臂停止或保持
3. 设置系统故障位
4. 上报 EVENT_HEARTBEAT_TIMEOUT
```

### CHASSIS_MOVE_REL

用途：相对运动，例如前进、后退、平移、原地旋转。

Payload 定长：

```text
Length = 19
```

| 偏移 | 字段          | 类型       | 单位      | 长度 |
| -: | ----------- | -------- | ------- | -: |
|  0 | dx_mm       | int32_t  | mm      |  4 |
|  4 | dy_mm       | int32_t  | mm      |  4 |
|  8 | dyaw_cdeg   | int32_t  | 0.01°   |  4 |
| 12 | vmax_mm_s   | uint16_t | mm/s    |  2 |
| 14 | wmax_cdeg_s | uint16_t | 0.01°/s |  2 |
| 16 | timeout_ms  | uint16_t | ms      |  2 |
| 18 | move_flags  | uint8_t  | -       |  1 |

`move_flags`：

|  bit |     掩码 | 含义          |
| ---: | -----: | ----------- |
| bit0 | `0x01` | 相对车体坐标      |
| bit1 | `0x02` | 相对世界坐标      |
| bit2 | `0x04` | 到位后停止       |
| bit3 | `0x08` | 启用 OPS 闭环   |
| bit4 | `0x10` | 启用 IMU 航向闭环 |
| bit5 | `0x20` | 允许误差范围内提前完成 |
| bit6 | `0x40` | 到位后保持刹车     |

示例：

```text
前进 1m：
dx_mm = 1000
dy_mm = 0
dyaw_cdeg = 0

后退 0.5m：
dx_mm = -500
dy_mm = 0
dyaw_cdeg = 0

原地左转 90°：
dx_mm = 0
dy_mm = 0
dyaw_cdeg = 9000
```

执行流程：

```text
1. STM32 收到 MOVE_REL
2. 检查 Length == 19
3. 检查参数范围
4. 返回 ACK_ACCEPTED
5. app_chassis 设置运动目标
6. 运动完成后上报 EVENT_MOTION_DONE
7. 超时或故障上报 EVENT_MOTION_FAILED
```

### ARM_SERVO_GROUP_SET

用途：多个舵机同步控制。

Payload 变长。

格式：

```text
uint8_t count
重复 count 次：
    uint8_t servo_id
    uint16_t position
    uint16_t speed
uint16_t time_ms
```

长度计算：

```text
Length = 1 + count * 5 + 2
```

该命令属于扩展变长命令，必须检查：

```c
if (Length != 1 + count * 5 + 2) return BAD_LENGTH;
```

### ARM_ACTION

用途：高级机械臂动作，例如上升、下降、夹取、放置。

Payload 定长：

```text
Length = 10
```

| 偏移 | 字段           | 类型       | 说明   | 长度 |
| -: | ------------ | -------- | ---- | -: |
|  0 | action_id    | uint8_t  | 动作编号 |  1 |
|  1 | param1       | int16_t  | 参数1  |  2 |
|  3 | param2       | int16_t  | 参数2  |  2 |
|  5 | speed        | uint16_t | 动作速度 |  2 |
|  7 | timeout_ms   | uint16_t | 超时   |  2 |
|  9 | action_flags | uint8_t  | 标志位  |  1 |

`action_id`：

|      值 | 名称            | 含义     |
| -----: | ------------- | ------ |
| `0x01` | ARM_HOME      | 回零     |
| `0x02` | ARM_UP        | 上升     |
| `0x03` | ARM_DOWN      | 下降     |
| `0x04` | GRIP_OPEN     | 夹爪张开   |
| `0x05` | GRIP_CLOSE    | 夹爪闭合   |
| `0x06` | PICK          | 执行夹取序列 |
| `0x07` | PLACE         | 执行放置序列 |
| `0x08` | ARM_STOP      | 停止     |
| `0x09` | ARM_TO_PRESET | 到预设姿态  |

`action_flags`：

|  bit |     掩码 | 含义            |
| ---: | -----: | ------------- |
| bit0 | `0x01` | 动作后保持力矩       |
| bit1 | `0x02` | 动作后释放力矩       |
| bit2 | `0x04` | 检查限位          |
| bit3 | `0x08` | 检查夹取成功        |
| bit4 | `0x10` | 动作完成后上报 EVENT |

示例：夹取动作。

```text

param1 = 100      // 高度或等级
param2 = 60       // 夹取力度

```

STM32 内部执行：

```text
1. 机械臂下降
2. 夹爪闭合
3. 延时或检测
4. 机械臂上升
```

解析原则：

```text

2. 允许粘包
3. 允许半包
4. CRC 错误丢弃

6. Payload 不在中断中执行
```

# 17. STM32 接收流程

```text
USART6 DMA + IDLE
    ↓
JetsonProtocol_OnRxData(uint8_t *data, uint16_t len)
    ↓
Protocol_InputBytes(data, len)
    ↓
状态机解析完整帧
    ↓
CRC16 检查
    ↓
Protocol_OnFrame(frame)
    ↓
App_DispatchCommand(frame)
    ↓
app_chassis / app_arm / app_safety
```

中断里禁止：

```text
1. HAL_Delay()
2. 大量 printf()
3. 控制电机运动
4. 控制机械臂动作
5. 阻塞等待 ACK
```

# 18. STM32 命令分发逻辑

```c
void App_DispatchCommand(const ProtocolFrame_t *frame)
{
    if (frame->frame_type != FRAME_CMD)
        return;

    switch (frame->cmd_set)
    {
    case CMDSET_SYSTEM:
        App_HandleSystemCommand(frame);
        break;

    case CMDSET_SAFETY:
        App_HandleSafetyCommand(frame);
        break;

    case CMDSET_CHASSIS:
        App_HandleChassisCommand(frame);
        break;

    case CMDSET_ARM:
        App_HandleArmCommand(frame);
        break;

    case CMDSET_CONFIG:
        App_HandleConfigCommand(frame);
        break;

    default:
        Protocol_SendAck(frame, ACK_UNKNOWN_CMD, ERR_UNKNOWN_CMD);
        break;
    }
}
```

# 20. 安全规则

必须实现：

```text
1. 上电默认底盘停止
2. 上电默认机械臂不执行动作
3. 300ms 未收到 Jetson 心跳，底盘急停
4. ESTOP 可打断任何动作
5. 运动类命令必须带 timeout_ms
6. CRC 错误帧直接丢弃
7. Length 错误帧返回 BAD_LENGTH
8. 当前模式不允许执行的命令返回 MODE_NOT_ALLOWED
9. 急停后必须 CLEAR_FAULT 才能恢复运动
10. 中断中不执行任何耗时动作
```

# 21. 与张大头步进电机协议的关系

上下位机协议和张大头电机协议不是一回事。

关系如下：

```text
Jetson
  ↓ 上下位机协议
STM32 jetson_protocol
  ↓ 命令分发
app_chassis
  ↓ 底盘控制
zdt_stepper
  ↓ 张大头原生协议
张大头步进电机扩展板
```

Jetson 不应该直接发张大头原生命令：

```text
01 F6 00 00 64 0A 00 6B
```

Jetson 应该发：

```text
CHASSIS_SET_MOTOR_RPM
CHASSIS_MOVE_REL
CHASSIS_GOTO_POSE
```

然后 STM32 内部再调用 `zdt_stepper.c`。这样后续换电机、换驱动器、改同步逻辑，都不影响 Jetson 侧协议。

# 24. 最终总结

本协议对 STM32 端的核心要求是：

```text
1. 外层使用 0x5A 0xA5 双帧头
2. 使用 Length 确定数据长度
3. 使用 CRC16 保证数据可靠
4. 核心命令 Payload 定长
5. 配置类命令允许变长
6. 使用 CmdSet + CmdID 支持扩展
7. 使用 Seq 关联 ACK / EVENT
8. 使用 HEARTBEAT 实现失联急停
9. 使用 STATUS 周期上报状态
10. 使用状态机解析串口字节流
```

这套协议适合你当前的 STM32F407 下位机结构，也能支撑后续加入 OPS、IMU、舵机、机械臂、预设点位和任务序列。
